FOTA Integration Guide

This guide describes the required steps to enable FOTA (Firmware Over-The-Air) services in your platform integration. Implementing the mandatory APIs ensures proper communication and operation of the FOTA agent with your device.

Required Implementation

Source File:
Source Files\config\default\library\kta_lib\SOURCE\kta\modules\fota_services\Fota_Platform.c

APIs to Implement:

  • fotaPlatformGetComponents()
  • fotaStartInstallation()

API Details

1. fotaPlatformGetComponents()

TKFotaStatus fotaPlatformGetComponents
(
TComponent *xpComponents
);

Purpose:
Called by the FOTA agent to obtain a list of all components and their versions currently installed on the device.

Integrator Responsibility:

  • Populate the xpComponents array with the relevant component information.
  • Ensure the array is not NULL.
  • Return the appropriate status code.

2. fotaStartInstallation()

void fotaStartInstallation
(
const void *xpFotaMetadata,
size_t xFotaMetadataLen,
const TTargetComponent *xpComponents
);

Purpose:
Initiates the installation process for the provided components.

Integrator Responsibility:

  • Start a dedicated FOTA thread or task to handle the installation.
  • Store component information securely so it can be recovered after a reboot.
  • Ensure the installation process is robust against interruptions.
  • After each component is installed, call fotaUpdateComponent() to inform the FOTA Agent.

FOTA Storage API

Source File:
Source Files\config\default\library\kta_lib\SOURCE\salapi\k_sal_fotastorage.c

APIs to Implement:

  • salFotaStorageWrite()
  • salFotaStorageRead()

Header File:
SOURCE\salapi\include\k_sal_fotastorage.h

Purpose:
These APIs are required for storing FOTA credentials and component data in flash storage. Use the MCU-specific, storage ID defined in k_sal_fotastorage.h.

Integrator Responsibility:

  • Implement persistent storage for FOTA credentials and component data.
  • Ensure data can be reliably written and read from flash storage.

Integration Notes

  • All listed APIs must be implemented for FOTA services to function correctly.
  • Ensure persistent storage of component data to support recovery after unexpected resets.
  • Refer to the FOTA agent documentation for additional integration requirements and best practices.
